      Information & Guide

      How to use LocalScrape

      1. Global Ingestion Modal: Click the "➕ Add Content" button in the header. From here, you can seamlessly queue up Article URLs, configure RSS Feeds, or securely bulk-upload EPUB/PDF books into your Bookshelf.

      📡 Settings Dashboard

      Manage system preferences via the ⚙️ Settings dashboard:

      • RSS Polling & Retention: Adjust how often the system checks your feeds, and set auto-deletion policies for older articles to save disk space.
      • UI & TTS Defaults: Customize your reading experience. Select default themes, fonts, and native Text-to-Speech voices.
      • Global Cookies: Inject your cookies.txt to safely bypass soft paywalls.

      3. Multi-Select Batch Actions:

      Check the box next to multiple articles in any grid to spawn a floating Action Panel. You can mark them as Read/Unread, Delete them simultaneously, or click 🧵 Megathread to stitch them together into a single reading digest.

      4. Bookshelf & Categories:

      The Bookshelf natively renders your PDFs and perfectly converts EPUBs into clean Markdown. It seamlessly tracks your scroll progress and automatically organizes your library into distinct Category sections based on the tags you set during upload.

      5. Explorer View Toggles:

      Organize your library layouts using the view controls above the main grids. Switch between Tiles (large icons), Details (clean tabular rows), or List (compact rows) to match your workflow.

      Architecture

      Frontend: A purely framework-free Single Page Application built with Vanilla HTML, CSS, and JS. It uses marked.js for rendering Markdown and features a Progressive Web App (PWA) manifest.

      Backend API: Built with Python and FastAPI. It provides RESTful endpoints for queuing scrapes, bulk operations, and file management.

      Data Philosophy: Strict "no database" rule. All data lives in the local filesystem inside the downloads/ folder.

      Data Flows

      All data is maintained locally as flat files:

      • tags/: Folders containing .md markdown articles.
      • books/: Folders containing .pdf, .epub, and their `.meta.json` companions.
      • feeds.json: RSS feed configurations.
      • settings.json: Local configuration overrides.
      • favorites.json: Contains all Starred article paths.

      Constraints & Limitations

      • Storage: Images are not downloaded locally; articles rely on hot-linking to original image URLs.
      • Browser: The bundled Playwright Chromium instance adds ~400MB to the Docker container size.
