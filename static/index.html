Information & Guide

How to use LocalScrape

1. Add Articles/Links: Click the "Add Articles/Links" button in the sidebar. Paste one or more URLs into the text area. Select an existing tag from the dropdown, or type a new tag name. Click "Scrape" — articles are processed in the background and will appear in your Library under the chosen tag.

💡 Smart Auto-Tagging: If you leave the Tag input blank, LocalScrape will automatically scan the article's keywords or title to create a relevant folder name (or default to 'uncategorized').

📡 RSS Feeds & Settings Dashboard

Manage background automation and system preferences seamlessly using the ⚙️ Settings dashboard in the sidebar:

  • RSS Feeds: Add feeds via the "RSS Feeds" sidebar. The system will poll automatically based on your RSS Polling Interval setting.
  • Retention Policy: The Settings dashboard allows you to define strict retention spans (in days) for older RSS articles and system log histories to prevent storage bloat.
  • UI & TTS Defaults: Customize your reading experience globally. Select a default theme, typography, font size, and native Text-to-Speech (TTS) voice and speed preferences.
  • Global Cookies: Inject your Netscape-formatted cookies.txt safely into the Scraping Engine directly from the UI to bypass soft paywalls.

3. Multi-Select Batch Actions & Megathreads:

Inside any Tag or Library view, you can check the box next to multiple articles to spawn a floating Action Panel. From there, you can mark them all as Read/Unread, Delete them simultaneously, or click 🧵 Megathread to stitch them together into a single, continuous reading digest.

4. Reader Power-Ups:

  • Bionic Reading: Toggle the brain icon in the reader view to dynamically bold the first half of words, guiding your eyes for faster reading.
  • Teleprompter: Activate auto-scroll from the reader controls for hands-free reading.
  • Web Share Target (Mobile PWA): "Add to Homescreen" on your phone to install the Progressive Web App. You can then use your OS's native "Share" sheet to instantly push URLs from your mobile browser straight into LocalScrape's queue!

5. Activity Monitor & Data Analysis: Track background queue statuses, job failures (up to 3 retries), and log outputs via the Activity Monitor. Review library footprints, read ratios, and total reading time aggregations dynamically overlaid across the dashboard tiles in the "Data Analysis" tab.

Architecture

Frontend: A purely framework-free Single Page Application built with Vanilla HTML, CSS, and JS. It uses marked.js for rendering Markdown and features a Progressive Web App (PWA) manifest that natively supports the Web Share Target API on mobile devices.

Backend API: Built with Python and FastAPI. It provides RESTful endpoints for queuing scrapes, checking status, executing bulk Megathread digests, and managing files (CRUD operations on the downloads directory).

Scraping Pipeline (4-Stage Core):
1. Check cached version on archive.md.
2. Fallback to Accelerated Mobile Pages (AMP) via ?amp.
3. Fallback to direct fetch with spoofed Referer headers.
4. Final fallback: Playwright headless Chromium (images/fonts blocked for speed).

Data Flows

Queue System: An asyncio.Queue manages background jobs sequentially to avoid rate-limiting. Status is tracked via the /api/queue endpoint.

Persistence: The app enforces a strict "no database" rule. All data lives in the local filesystem inside the downloads/ folder:

  • tags/: Folders containing .md markdown articles.
  • feeds.json: RSS feed configurations.
  • failed.json: Vault of failed scrape attempts (max 3 strikes).
  • completed.json: Ledger of historical scrapes to block exact-match URL duplication.
  • read_state.json: Dictionary tracking boolean read/unread statuses.
  • settings.json: Local configuration overrides (e.g., Gemini API keys).
  • favorites.json: Contains all Starred article paths.

Constraints & Limitations

  • Full-Text Search: Performed via linear filesystem scanning. Not suitable for corpuses >10,000 articles (requires migrating to a TF-IDF index).
  • Storage: Images are not downloaded locally; articles rely on hot-linking to original image URLs.
  • Browser: The bundled Playwright Chromium instance adds ~400MB to the Docker container size.
